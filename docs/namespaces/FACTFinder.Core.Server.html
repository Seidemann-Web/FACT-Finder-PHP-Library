\FACTFinder\Core

\FACTFinder\CoreServer

Interfaces

RequestFactoryInterface Use this to obtain Request objects instead of creating them directly.

Classes

AbstractDataProvider Data providers take care of retrieving pre-configured requests with FACT-Finder data, including the network connection if necessary.
ConnectionData Represents all data belonging to a single connection. It holds all data necessary to configure a request and can also be filled with the corresponding response.
EasyCurlDataProvider This implementation retrieves the FACT-Finder data by using the "easy cURL interface" (I didn't even make that up; that's what cURL itself calls it: 'easy' as opposed to 'multi').
EasyCurlRequestFactory This implementation backs the Request with an EasyCurlDataProvider.
FileSystemDataProvider This implementation retrieves the FACT-Finder data from the file system. File names are generated from request parameters. For the naming convention see the getFileName() method.
FileSystemRequestFactory This implementation backs the Request with a FileSystemDataProvider.
MultiCurlDataProvider This implementation retrieves the FACT-Finder data by using the cURL's "multi interface".
MultiCurlRequestFactory This implementation backs the Request with a MultiCurlDataProvider.
NullResponse Null Object implementation of the Response class, e.g. to be used if no response could be obtained.
Request This is basically a Facade for ConnectionData and DataProviders, allowing access only to parameters, HTTP header fields, the target action a few selected connection options, plus retrieval of the response. ConnectionData and DataProvider are hidden behind this class and should not be used by anyone except Requests and their factories.
Response Represents a request response from the FACT-Finder server.
UrlBuilder Assembles URLs to the FACT-Finder server for different kinds of authentication based on the given parameters and the configuration.
